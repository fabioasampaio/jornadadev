Exercício 1 — Isto é um algoritmo válido? Para cada situação abaixo, indique se é um algoritmo válido. Se não for, explique por quê (lembre dos três requisitos: finito, ordenado e sem ambiguidade). 

a. “Some dois números.” 
* Não é um algoritmo válido. A instrução é ambígua, pois não informa quais são os dois números nem se eles devem ser lidos como entrada. Falta detalhamento dos passos.

b. “Leia um número. Se for positivo, mostre ‘positivo’. Senão, mostre ‘negativo’.” 
* É um algoritmo válido. Possui uma sequência de passos (ordenado), termina após exibir o resultado (finito) e as instruções são claras (sem ambiguidade).

c. “Fique repetindo até acabar.” 
* Não é um algoritmo válido. A condição "até acabar" é ambígua, pois não informa o que deve acabar. Além disso, não há garantia de que o algoritmo seja finito.

d. “Pegue um ingrediente qualquer e cozinhe.” 
* Não é um algoritmo válido.É ambíguo, pois não especifica qual ingrediente deve ser utilizado nem como cozinhá-lo (tempo, temperatura, modo de preparo etc.). As instruções não são precisas
